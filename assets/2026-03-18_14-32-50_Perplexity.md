## Граф в vault.typ через diagraph

Есть пакет **diagraph** который использует Graphviz для рендеринга графов:[typst](https://typst.app/universe/package/diagraph-layout/)​

text

// vault.typ
#import "@preview/diagraph:0.3.0": *

// ... определения типов ...

// Генерация графа связей
#let build-graph() = {
 let paths = csv("note-paths.csv").flatten()
 let nodes = ()
 let edges = ()
 
 // Парсим все заметки
 for path in paths {
 let content = read(path)
 let ast = base.parse(content)
 let meta = base.extract-metadata(ast)
 let links = base.extract-links(ast) // найти все xlink
 
 nodes.push(meta.title)
 
 for link-id in links {
 edges.push((meta.id, link-id))
 }
 }
 
 // Рисуем граф
 raw-render(```
 digraph {
 ${nodes.map(n => n).join("\n")}
 ${edges.map(e => e.at(0) + " -> " + e.at(1)).join("\n")}
 }
 ```)
}

// В конце vault.typ
#pagebreak()
= Notes Graph
#build-graph()

#import "@preview/cetz:0.3.1": canvas, draw

#let build-graph() = {
 canvas({
 import draw: *
 
 // Получаем заметки и связи
 let notes = get-all-notes()
 
 // Force-directed layout (упрощенно)
 for (i, note) in notes.enumerate() {
 let x = calc.cos(i * 360deg / notes.len()) * 3
 let y = calc.sin(i * 360deg / notes.len()) * 3
 
 // Рисуем узел
 circle((x, y), radius: 0.3, 
 fill: blue.lighten(50%),
 name: note.id
 )
 content((x, y), note.title)
 
 // Рисуем связи
 for link in note.links {
 line(note.id, link, mark: (end: ">"))
 }
 }
 })
}

## Архитектура Typst-части

**База: пакет `@local/notes` + корневой `vault.typ`.**

`@local/notes` — стабильная библиотека:

функции для создания метаданных заметок (типизированные конструкторы);

парсинг других `.typ` файлов через AST;

примитивы для query/fильтрации и построения графа связей;

вспомогательные штуки для backlinks и форматирования.

`vault.typ` (лежит в корне “хранилища заметок”):

импортирует `@local/notes`;

читает `note-paths.csv` со списком `.typ` файлов;

один раз (через кеш/state) парсит все заметки, вытаскивает метаданные и ссылки;

объявляет “типовые” конструкторы заметок (task, note, tag, card, пользовательские типы);

задаёт форматтеры (единый стиль, поведение root/branch-заметок);

в конце документа рендерит:

автоматический блок backlinks для root-заметок;

граф связей заметок (через diagraph/CeTZ-подобный подход).

Все отдельные заметки импортируют **только** `vault.typ`, а не напрямую пакет.

## Функции фреймворка Typst

Условные ключевые API (семантика, а не строгий синтаксис):

`vault.new-note-type(type: str, fields: dict)` 

Определяет новый тип заметки (например, `task`, `note`, `card`, `tag`, `recipe`). 

Возвращает функцию-конструктор, у которой явно видны параметры (для автодополнения).

Сгенерированные конструкторы:

`task(id, title, due_date, tags, ...)`

`note(id, title, content, ...)`

`card(id, front, back, ...)`

`tag(id, name, color, ...)`

и любые пользовательские типы, объявленные в `vault.typ`.

Внутри создают `metadata(...)` с полями `id`, `type`, `title`, тэгами и т.д.

`vault.query-notes(type: none, where: (note) => bool, sort-by: none)` 

Работает поверх кешированного индекса (в памяти) по всем файлам из `note-paths.csv`. 

Возвращает массив структур вида `{ id, type, title, tags, path, … }`.

`vault.xlink(id: str)` / `vault.xlink(name: str)` 

Логическая ссылка на другую заметку:

если цель включена как branch в текущий документ — внутренняя ссылка;

если нет — ссылка на отдельный PDF/HTML заметки.

`vault.render-backlinks()` 

Внутренняя реализация блока “Backlinks”, который вызывается автоматически форматтером для root-заметок (в конце документа).

`vault.build-graph()` 

Строит структуру узлов и рёбер (id, title, links), после чего вызывает визуализацию (через diagraph/SVG-диаграмму) и рисует граф связей.

Вспомогательные функции внутри базы:

`parse(content)` — обёртка над AST-парсером Typst;

`extract-metadata(ast)` — вытаскивает `id`, `title`, `type`, теги и т.п.;

`extract-links(ast)` — находит все вызовы `xlink(...)` и формирует список ссылок;

кеширование индекса через `state`/`context`, чтобы за одну компиляцию всё парсилось только один раз.

## Как всё вместе работает

Пользователь создаёт заметку из приложения:

backend создаёт `.typ` с вызовом нужного конструктора (например, `#show: task.with(...)`);

добавляет этот файл в `note-paths.csv`.

При работе с заметкой:

редактор (через LSP) даёт автодополнение по функциям фреймворка;

пользователь может использовать Cmd+K для поиска и вставки `#xlink`.

При компиляции заметки:

Typst выполняет `vault.typ`, который:

читает `note-paths.csv`;

через базовый модуль один раз парсит все заметки, строит индекс в памяти;

предоставляет конструкторы типов, query-функции и xlink;

форматтеры добавляют в конец блока заметки список backlinks.

Для обзора / графа:

пользователь открывает/компилирует `vault.typ`;

в конце документа рендерится граф связей (узлы — заметки, рёбра — ссылки);

это работает без специальных фич в приложении, т.к. логика графа и визуализации сидит в Typst.

## Важные принципы, которые выстроились

**Vault как центр системы**: все заметки зависят от одного `vault.typ`, а он — от одного `@local/notes`.

**Ручное редактирование типов**: типы определяются прямо в `vault.typ`, без скрытого JSON — всё видно и управляется как код.

**Меньше внешних артефактов**: `note-paths.csv` обязателен, индекс JSON — опционален (добавляется только при необходимости оптимизации).
